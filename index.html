This page demonstrates a maze-solving cellular automata ruleset I created. It can find the most efficient solution to any maze, even ones with many complicated loop structures. It uses 8 different cell states: two to represent the start and finish, two to define the maze, and four to handle the actual solving process. Dead-ends in the maze are handled by the well-known end-filling procedure. Loops are handled in a somewhat more novel way: an expanding "slime" bounces back and forth from the start to the finish. The slower routes around loops get killed off, leaving only the fastest ones. If two routes are equally efficient, both will be included in the CA's final solution

I decided to create this ruleset after viewing images of slime mold solving mazes. Slime mold can solve a maze if the start and finish are presented to it as two piles of food. The mold, which prefers to remain a unified system, will stretch itself between the two in the shortest way possible. Though this cellular automata system works in a different fashion, both demonstrate that simple, decentralized system can solve challenging problems.
